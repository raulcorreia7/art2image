# 03 Simple CHANGELOG

## Intent
Basic changelog for release notes.

## File
`CHANGELOG.md` with standard format:
```markdown
# Changelog

## [Unreleased]
- Changes

## [1.0.0] - Date
- Initial release
```

## Integration
- Update during version bump
- Include in release notes

## Acceptance
- CHANGELOG.md exists
- Updated during releases
- Used in GitHub releases
